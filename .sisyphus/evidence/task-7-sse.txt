Task 7 — SSE Streaming Format Evidence
========================================

Endpoint: POST /api/chat
Content-Type: text/event-stream; charset=utf-8

SSE Format (verified via pytest integration tests):
---------------------------------------------------

Each chunk follows the SSE spec:
  data: <json>\n\n

Token chunks:
  data: {"token": "Hello", "done": false}\n\n
  data: {"token": " ", "done": false}\n\n
  data: {"token": "world", "done": false}\n\n
  data: {"token": "!", "done": false}\n\n

Final done chunk:
  data: {"token": "", "done": true}\n\n

Error chunk:
  data: {"error": "Internal server error", "done": true}\n\n

Verification Tests Passed:
--------------------------
  test_sse_content_type           — asserts content-type header
  test_sse_stream_format          — asserts every chunk has "data: " prefix
  test_sse_ends_with_done_true    — asserts final chunk {"token":"","done":true}
  test_sse_tokens_in_order        — asserts tokens match service output order

Implementation:
---------------
File: backend/app/api/chat.py
  - ChatService is lazy-initialized (module-level var, created on first request)
  - event_generator() wraps stream_chat() in SSE format
  - Exceptions caught and returned as SSE error chunks
  - No GET-based SSE; only POST /api/chat
